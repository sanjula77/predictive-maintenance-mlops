# Contributing Guidelines

Thank you for your interest in contributing to Predictive Maintenance MLOps!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a virtual environment
4. Install dependencies: `pip install -r requirements.txt`
5. Install development dependencies: `pip install -r requirements-dev.txt` (if exists)

## Code Style

- Follow PEP 8 style guide
- Use type hints where possible
- Write docstrings for all functions and classes
- Keep functions focused and small

## Testing

- Write tests for new features
- Run tests: `pytest tests/`
- Ensure all tests pass before submitting PR

## Pull Request Process

1. Create a feature branch
2. Make your changes
3. Add/update tests
4. Update documentation if needed
5. Submit a pull request with a clear description

## Commit Messages

Use clear, descriptive commit messages:
- `feat: Add new model architecture`
- `fix: Fix data loading bug`
- `docs: Update README`
- `refactor: Clean up code structure`
